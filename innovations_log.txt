================================================================================
ЖУРНАЛ ИННОВАЦИЙ И ОБНОВЛЕНИЙ AI CONTOUR
Период: с момента последнего git pull до коммита 8be42f4
================================================================================

В рамках проведенной работы над кодовой базой AI Contour (Итерация 4.5 и Итерация 5) 
были реализованы следующие ключевые нововведения, исправления багов и улучшения UX/безопасности:

--------------------------------------------------------------------------------
1. ВИЗУАЛЬНЫЙ ТЮНИНГ И UX УЛУЧШЕНИЯ (Итерация 4.5)
--------------------------------------------------------------------------------
* Бирюзово-мятный прогресс-бар:
  Внедрен премиальный линейный полосатый градиент с оттенками #00bda6 и #00a38f 
  и закруглением border-radius: 4px, улучшающий визуальную эстетику.
* Чистка чек-боксов отображения структур:
  Удалены жесткие инлайн-стили чек-боксов. Теперь индикатор наследует премиальный 
  глубокий синий цвет (#007acc) из глобальной темы DARK_QSS, а текст остается белым.
* Сохранение размеров окон просмотра при переключении пациентов:
  Сплиттеры окон просмотра больше не сжимаются при смене DICOM-серий, сохраняя 
  увеличенный размер и предотвращая "прыжки" интерфейса.
* [БАГФИКС] Устранение рассинхронизации булевой маски:
  Исправлена критическая ошибка "boolean index did not match indexed array along axis 0". 
  Отрисовка масок перенесена на момент, когда КТ-массив вьюера полностью загружен.
* [UX] Стабилизация размеров окон:
  Настройка сплиттеров на увеличенный масштаб перенесена в слот ручного включения 
  галочки, убирая автоматические сбросы размеров при автосканировании.

--------------------------------------------------------------------------------
2. БЕЗОПАСНОСТЬ И УПРАВЛЕНИЕ КЛИЕНТАМИ (Итерация 5)
--------------------------------------------------------------------------------
* Блокировка по Client ID (DHCP Safety):
  Вместо ненадежной привязки к динамическим IP-адресам (DHCP), блокировка ПК 
  производится по уникальному текстовому Client ID (Имени ПК), который клиент 
  передает в заголовке "X-Client-ID" при каждом запросе.
* FastAPI Middleware и BlacklistManager:
  На стороне сервера внедрено ClientTrackerMiddleware и BlacklistManager для 
  автоматического перехвата заголовков, регистрации активности и возврата 
  403 Forbidden заблокированным клиентам.
* Скрытая панель администратора:
  Параметры соединения и лицензирования ИИ на сервере объединены в единую панель 
  admin_settings_widget, скрытую по умолчанию.
* Авторизация по паролю "rtp":
  Доступ к панели настроек администратора открывается только при успешном вводе 
  пароля "rtp" в диалоговом окне авторизации.
* Окно мониторинга ConnectedClientsDialog:
  Создано интерактивное окно со списком подключенных ПК, обновляющееся по таймеру 
  каждые 3 секунды, позволяющее блокировать и разблокировать клиентов в реальном времени.
* Анимация кнопки выхода из превью:
  Кнопке выхода задан премиальный оранжевый градиент с QSS-анимацией наведения/нажатия. 
  Реализована динамическая смена objectName ("btnRun" / "btnExitPreview") 
  с принудительной полировкой стиля в Qt.

--------------------------------------------------------------------------------
3. ИСПРАВЛЕНИЕ УЯЗВИМОСТЕЙ БЕЗОПАСНОСТИ (Отчет Линтера)
--------------------------------------------------------------------------------
* Устранение OS Command Injection (CWE-78):
  Все вызовы утилиты "taskkill" в server_app.py переведены с небезопасного shell=True 
  и конкатенации строк на безопасный список аргументов с флагом shell=False.
* Устранение Risky Cryptographic Algorithm (CWE-327):
  Заменили устаревший хэш-алгоритм hashlib.md5 на надежный hashlib.sha256 для 
  генерации дефолтных цветов OAR-структур.

--------------------------------------------------------------------------------
4. УЛУЧШЕНИЯ UX СТАТУСОВ СЕТИ И КНОПКИ ЗАПУСКА
--------------------------------------------------------------------------------
* Динамический статус "Офлайн" для клиентов:
  Если клиент закрыл приложение (или потерял связь) и не отправлял пинги более 10 секунд, 
  сервер переводит его в статус "Офлайн", а в окне мониторинга строка подсвечивается 
  нейтральным серым цветом (#888888).
* Устранение мигания кнопки "Отправить в очередь":
  Убрана принудительная блокировка и "затемнение" кнопки запуска при автоматическом 
  фоновом сканировании папок DICOM (происходящем каждые 15 секунд). Кнопка блокируется 
  только при ручном сканировании.
* Интеллектуальная блокировка кнопки:
  Кнопка на клиенте блокируется и выводит наглядные статусы при сбое сети:
  - При блокировке ПК: "ДОСТУП ЗАБЛОКИРОВАН ❌"
  - При выключенном сервере: "СЕРВЕР ОФЛАЙН 🔌"
  - При паузе очереди на сервере: "СЕРВЕР НА ПАУЗЕ ⏸️"
